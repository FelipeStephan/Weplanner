# WePlanner — PRD Base

> Documento de referência estratégica. Escrito na perspectiva de Principal Product Designer & SaaS Strategist.

---

## O que é este diretório

Esta pasta contém a fundação de produto da WePlanner — a base que orienta todas as decisões de design, engenharia e go-to-market. Não é documentação técnica (essa vive em `/docs`). É a **visão de produto**: o porquê, o para quem, e o como de forma intencional.

## Como usar estes documentos

- **Para decisões de roadmap:** leia `01-positioning.md` primeiro. Toda feature deve se conectar ao posicionamento.
- **Para design de novas features:** leia `02-profile-architecture.md`. Quem usa isso? Em qual contexto?
- **Para refinamento de produto existente:** leia `03-core-features.md`. Como cada módulo conecta ao valor central?
- **Para decisões de backend e dados:** leia `04-data-flow-governance.md`. Quem é dono do quê?

## Princípio editorial deste PRD

> "Uma plataforma de categoria se define não pelo que faz, mas pelo problema que elimina para sempre da vida do usuário."

WePlanner não é uma ferramenta de tarefas. É a **infraestrutura operacional de agências** que transforma caos de execução em clareza de entrega — com rastreabilidade de valor para cada cliente.
